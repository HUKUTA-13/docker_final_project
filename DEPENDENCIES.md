# Зависимости CRM Basic

Этот документ содержит полный список всех зависимостей, используемых в проекте CRM Basic.

## 🖥️ Системные требования

### Операционная система
- Linux (Ubuntu, Debian, Fedora, etc.)
- macOS
- Windows с WSL2

### Программное обеспечение
- **Node.js**: версия 18.x или выше
- **npm**: версия 8.x или выше (поставляется с Node.js)
- **PostgreSQL**: версия 12.x или выше

## 📦 Backend зависимости

### Основные зависимости (dependencies)

#### express (^4.18.2)
- **Описание**: Минималистичный веб-фреймворк для Node.js
- **Назначение**: Создание REST API, обработка HTTP запросов
- **Документация**: https://expressjs.com/

#### pg (^8.11.0)
- **Описание**: PostgreSQL клиент для Node.js
- **Назначение**: Подключение к базе данных PostgreSQL, выполнение SQL запросов
- **Документация**: https://node-postgres.com/

#### cors (^2.8.5)
- **Описание**: Middleware для Express для настройки CORS
- **Назначение**: Разрешение кросс-доменных запросов от Frontend к Backend
- **Документация**: https://github.com/expressjs/cors

### Зависимости для разработки (devDependencies)

#### nodemon (^2.0.22)
- **Описание**: Утилита для автоматической перезагрузки Node.js приложения
- **Назначение**: Hot-reload при разработке, автоматический перезапуск сервера при изменении кода
- **Документация**: https://nodemon.io/

## 🎨 Frontend зависимости

### Основные зависимости (dependencies)

#### react (^18.2.0)
- **Описание**: JavaScript библиотека для создания пользовательских интерфейсов
- **Назначение**: Основа Frontend приложения, компонентный подход
- **Документация**: https://react.dev/

#### react-dom (^18.2.0)
- **Описание**: React пакет для работы с DOM
- **Назначение**: Рендеринг React компонентов в браузере
- **Документация**: https://react.dev/reference/react-dom

#### react-scripts (5.0.1)
- **Описание**: Скрипты и конфигурация Create React App
- **Назначение**: Сборка, запуск dev-сервера, настройка webpack
- **Документация**: https://create-react-app.dev/

#### typescript (^4.9.5)
- **Описание**: Типизированный надстройка над JavaScript
- **Назначение**: Статическая типизация, улучшение качества кода
- **Документация**: https://www.typescriptlang.org/

#### axios (^1.6.7)
- **Описание**: HTTP клиент на основе промисов
- **Назначение**: Выполнение HTTP запросов к Backend API
- **Документация**: https://axios-http.com/

#### http-proxy-middleware (^2.0.6)
- **Описание**: Middleware для проксирования HTTP запросов
- **Назначение**: Проксирование API запросов в режиме разработки
- **Документация**: https://github.com/chimurai/http-proxy-middleware

### Типы для TypeScript

#### @types/react (^18.2.0)
- **Описание**: TypeScript определения типов для React
- **Назначение**: Типизация React компонентов и хуков

#### @types/react-dom (^18.2.0)
- **Описание**: TypeScript определения типов для React DOM
- **Назначение**: Типизация методов react-dom

## 🗄️ База данных

### PostgreSQL
- **Версия**: 12.x или выше
- **Назначение**: Реляционная база данных для хранения информации о клиентах
- **Особенности использования**:
  - Используется стандартный порт 5432
  - Пользователь: postgres
  - База данных: crm_db

## 🔧 Установка всех зависимостей

### Автоматическая установка
Запустите скрипт `start.sh`, который автоматически установит все необходимые npm пакеты:
```bash
./start.sh
```

### Ручная установка

#### Backend:
```bash
cd backend
npm install
```

#### Frontend:
```bash
cd frontend
npm install
```

## 🔄 Обновление зависимостей

Для проверки устаревших пакетов:
```bash
# Backend
cd backend && npm outdated

# Frontend
cd frontend && npm outdated
```

Для обновления зависимостей:
```bash
# Обновить до последних минорных версий
npm update

# Обновить до последних мажорных версий (осторожно!)
npm install package-name@latest
```

## ⚠️ Известные проблемы

1. **react-scripts 5.0.1** - может выдавать предупреждения о устаревших зависимостях, но это не влияет на работу приложения

2. **PostgreSQL** - убедитесь, что сервис запущен и доступен на порту 5432

## 🔒 Безопасность

Регулярно проверяйте зависимости на уязвимости:
```bash
npm audit
```

Для автоматического исправления:
```bash
npm audit fix
```
